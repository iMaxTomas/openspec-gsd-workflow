  我先判断这一轮到底要什么
  -> 是先把事情说清楚
  -> 还是按计划往前做
  -> 还是把推进动作交给脚本
  -> 还是先停下来审计“到底做完没有”


                         not this : 一开口就默认“AI 全自动把整个项目做完”
                         but this : 先定 mode，再让 AI 扮演 planner / copilot / operator / auditor

|                                                四句最短判断                                                         |
| 方式1 = 先把“要做什么”说清楚                                                                                        |
| 方式2 = 边做边手动管 context / checkpoint / handoff                                                                 |
| 方式3 = 把 branch / pause / worktree / resume 这条机械链路自动化                                                   |
| 方式4 = 不往前冲，先核对“是不是按 OpenSpec 真做完了”                                                                |
+--------------------------------------------------------------------------------------------------------------------+


+--------------------------------------------------------------------------------------------------------------------+
|                                                一眼怎么选                                                           |
| 需求还模糊                     -> 方式1                                                                             |
| 方案已定，但我想自己控节奏     -> 方式2                                                                             |
| 流程重复，想把机械动作交出去   -> 方式3                                                                             |
| 团队口头说“做完了”，我不放心   -> 方式4                                                                             |
